  **ProfileApi.php line 245 — Profile::query() does not exist:**
  
  The `get_profiles()` method (lines 237-275) uses Eloquent-style `Profile::query()` which was removed during the migration to WordPress-native storage. Rewrite it to use `Profile::get_all()` with WordPress-native query args.
  
  Replace the method body with logic that:
  1. Uses `Profile::get_all()` to fetch profiles
  2. Applies search filter by iterating results (matching first_name, last_name, email, nmls against search term)
  3. Applies status filter by checking `is_active`

  Simplified approach since this is an admin endpoint and Profile::get_all() already returns hydrated Profile objects:
  ```php
  public function get_profiles( $request ) {
      $page     = (int) ( $request->get_param( 'page' ) ?: 1 );
      $per_page = (int) ( $request->get_param( 'per_page' ) ?: 20 );
      $search   = $request->get_param( 'search' );
      
      $profiles = Profile::get_all();
      
      // Convert to arrays
      $results = array_map( function( $p ) {
          return $p->toArray();
      }, $profiles );
      
      // Search filter
      if ( $search ) {
          $search = strtolower( $search );
          $results = array_filter( $results, function( $p ) use ( $search ) {
              $haystack = strtolower(
                  ( $p['first_name'] ?? '' ) . ' ' .
                  ( $p['last_name'] ?? '' ) . ' ' .
                  ( $p['email'] ?? '' ) . ' ' .
                  ( $p['nmls'] ?? '' )
              );
              return strpos( $haystack, $search ) !== false;
          });
      }
      
      $total   = count( $results );
      $results = array_values( $results );
      $results = array_slice( $results, ( $page - 1 ) * $per_page, $per_page );
      
      $response = new \WP_REST_Response( $results, 200 );
      $response->header( 'X-WP-Total', $total );
      $response->header( 'X-WP-TotalPages', ceil( $total / $per_page ) );
      
      return $response;
  }
  ```
