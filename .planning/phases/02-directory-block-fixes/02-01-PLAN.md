<objective>
Fix the QR popup so it works on all cards, including those rendered by JavaScript after search/filter/load-more.

Root cause: `renderGrid()` uses `innerHTML` to create cards, which bypasses Interactivity API directives. The server-rendered cards have `data-wp-on-async--click="actions.openQrPopup"` but JS-created cards don't get these directives processed.

Fix approach: After `renderGrid()` creates cards via innerHTML, attach vanilla click event listeners to the QR buttons that call the same state mutations. This is the safest fix -- it doesn't require rewriting the entire rendering approach.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add click handlers to JS-rendered QR buttons</name>
  <files>assets/blocks/src/directory-grid/view.js</files>
  <action>
  In the `renderGrid()` action (line ~279), after `grid.appendChild(createCard(...))`, add a loop that finds all `.frs-card__qr-btn` elements in the grid and attaches click listeners that update state to open the QR popup.

  After the forEach loop that appends cards, add:
  ```js
  // Attach QR popup handlers to dynamically created cards
  grid.querySelectorAll('.frs-card__qr-btn').forEach((btn) => {
      btn.addEventListener('click', () => {
          const qrData = btn.dataset.qr;
          const name = btn.dataset.name;
          if (qrData) {
              state.qrImageSrc = qrData;
              state.qrName = name || '';
              state.qrPopupOpen = true;
          }
      });
  });
  ```

  This mirrors exactly what `actions.openQrPopup` does but works on vanilla DOM elements.
