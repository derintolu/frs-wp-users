### Generic Component Shortcode

#### `[lrh_component]` ‚ö°
**Generic React component renderer**
- Embed ANY React component from the component registry
- Pass props as JSON
- Custom wrapper styling

**Attributes:**

- `name` - Component name (required)
- `props` - JSON props object
- `wrapper_class` - CSS classes
- `wrapper_style` - Inline styles

**Available Components:**

- MyProfile, MarketingOverview, LeadTracking
- FluentBookingCalendar, LandingPagesMarketing, EmailCampaignsMarketing
- LocalSEOMarketing, BrandShowcase, MarketingOrders
- MortgageCalculator, PropertyValuation, Settings, MarketingSubnav
- RealtorOverview, MarketingTools, CalculatorTools, CompanyOverview, Resources

```
[lrh_component name="MortgageCalculator"]
[lrh_component name="MyProfile" props='{"userId":"123","autoEdit":true}']
[lrh_component name="LeadTracking" wrapper_class="p-4 bg-white"]
```

## üìã Usage in SureDash

### Best Practices for SureDash Integration

1. **Use content-only shortcodes** for embedding in SureDash pages:
   - `[frs_my_profile]`
   - `[frs_profile_settings]`
   - `[frs_welcome]`

2. **Create separate SureDash pages** for each content area:
   - Dashboard ‚Üí `[frs_welcome]`
   - My Profile ‚Üí `[frs_my_profile]`
   - Settings ‚Üí `[frs_profile_settings]`
   - Directory ‚Üí `[frs_profile_directory]`

3. **Let SureDash handle navigation** - Use content-only shortcodes and rely on SureDash's sidebar

4. **Combine with SureDash blocks** - Mix FRS shortcodes with SureDash identity, navigation, and content blocks

## üÜò Troubleshooting

### Shortcode Not Rendering

1. **Check plugin is active**: Go to Plugins ‚Üí Make sure the plugin is activated
2. **Clear cache**: Run `wp cache flush` or clear your caching plugin
3. **Check for conflicts**: Temporarily disable other plugins to test
4. **Review shortcode name**: Ensure correct spelling and underscores (not hyphens)

### Styling Issues

1. **Check theme compatibility**: Some themes may override styles
2. **Use content-only shortcodes in SureDash**: Avoid sidebar conflicts
3. **Add custom CSS**: Target shortcode containers with custom classes

## ‚ö° Performance Optimizations (December 2024)

### Bundle Size Improvements

All React-based shortcodes now benefit from comprehensive bundle optimizations:

**What was optimized:**

- ‚úÖ Automatic code splitting - Vendor libraries cached separately
- ‚úÖ Manual chunking - Heavy libraries (Radix UI, Recharts, React Router) in separate chunks
- ‚úÖ Terser minification - Better compression
- ‚úÖ Tree shaking - Unused code removed

**Results:**

- **Main bundles:** 70-92% smaller
- **Faster loading:** Reduced initial load time significantly
- **Better caching:** Vendor code cached across pages
- **Parallel downloads:** Multiple small chunks load simultaneously

**Affected shortcodes:**

- `[frs_profile]` - Portal bundle optimized
- `[frs_my_profile]` - Portal bundle optimized
- `[frs_profile_settings]` - Portal bundle optimized
- `[frs_welcome]` - Portal bundle optimized
- `[frs_profile_directory]` - Directory bundle optimized

### Loading Performance Tips

1. **Use content-only shortcodes** - Smaller bundles, faster loads
2. **Browser caching works better** - Vendor chunks cached across visits
3. **Mobile performance improved** - Smaller bundles = faster mobile loads
4. **SureDash integration optimized** - Code splits load only what's needed
